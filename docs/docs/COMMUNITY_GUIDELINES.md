# Community Guidelines

_Last updated: July 20, 2026_

## Overview

CeloHT is a community-governed open-source initiative built on collaboration, transparency, education, and shared responsibility.

These guidelines help create a respectful, inclusive, and productive environment for all community members.

---

# Community Values

The CeloHT community is guided by:

- Respect
- Inclusion
- Transparency
- Collaboration
- Learning
- Responsibility
- Sustainability

---

# Expected Behavior

Community members should:

- Treat others with respect.
- Communicate honestly.
- Share knowledge.
- Welcome different perspectives.
- Provide constructive feedback.
- Support new contributors.

---

# Unacceptable Behavior

The community does not tolerate:

- Harassment
- Discrimination
- Threats
- Personal attacks
- Spam
- Fraudulent activities
- Deliberate misinformation
- Abuse of community resources

---

# Communication Standards

Good communication should be:

- Clear
- Respectful
- Professional
- Helpful

When disagreeing:

- Discuss ideas, not people.
- Provide evidence when possible.
- Remain open to feedback.

---

# GitHub Participation

When using CeloHT repositories:

Members should:

- Search existing Issues before creating new ones.
- Provide clear descriptions.
- Respect review processes.
- Follow contribution guidelines.

---

# Discussions

GitHub Discussions should be used for:

- Ideas
- Questions
- Community proposals
- Knowledge sharing
- General conversations

Avoid:

- Repeated posts
- Unrelated promotion
- Harmful content

---

# Contributors

All contributors are encouraged to:

- Follow CONTRIBUTING.md.
- Respect project standards.
- Document their changes.
- Help improve the community.

---

# Privacy and Safety

Community members should:

- Respect personal information.
- Avoid sharing private data.
- Protect account security.
- Never request sensitive credentials.

---

# Leadership and Roles

Community roles exist to support collaboration.

Roles do not represent ownership or absolute authority over the community.

---

# Conflict Resolution

Conflicts should be handled through:

1. Respectful discussion.
2. Community moderation.
3. Mediation when necessary.
4. Application of relevant policies.

---

# Reporting Issues

Concerns about community behavior may be reported through the official CeloHT communication channels.

Reports should include:

- Description of the issue.
- Relevant details.
- Evidence when available.

---

# Updates

These guidelines may evolve through the CeloHT community governance process.
